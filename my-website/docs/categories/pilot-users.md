# Pilot Users

Feedback relacionado con la captación y gestión de usuarios piloto.

---

## ✅ Lo que funciona

- Segmentar correctamente a los usuarios piloto y estructurar su feedback permite obtener información útil para mejorar el producto.
- Incluir perfiles diversos (usuarios más técnicos y menos técnicos) ayuda a detectar problemas desde diferentes perspectivas.
- Definir claramente el ICP (Ideal Customer Profile) permite seleccionar usuarios piloto que representen mejor al público objetivo del producto.
- Describir una estrategia clara de captación de usuarios piloto mejora la calidad y representatividad del feedback obtenido.
- Incorporar usuarios piloto en fases tempranas del desarrollo permite validar el producto y detectar problemas antes de escalar.
- Establecer una estrategia progresiva para ampliar el número de usuarios piloto facilita validar el producto de forma controlada.
- Utilizar contacto directo con los usuarios piloto (entrevistas u observación) permite comprender mejor cómo interactúan realmente con la aplicación.
- Observar el comportamiento real de los usuarios mientras utilizan la aplicación proporciona información más valiosa que solo respuestas escritas.
- Mostrar ejemplos concretos de feedback de usuarios piloto y explicar qué cambios se han implementado a partir de él demuestra capacidad de aprendizaje y adaptación del equipo.
- Incorporar el tiempo necesario para procesar y analizar el feedback de usuarios piloto en la planificación del proyecto mejora la calidad de las iteraciones del producto.

---

## ❌ Lo que no funciona

- Limitar el feedback de usuarios piloto únicamente a formularios reduce la calidad de la información obtenida.
- No estructurar ni segmentar correctamente el feedback de los usuarios piloto dificulta su análisis y aplicación en el desarrollo del producto.
- Utilizar únicamente usuarios piloto con perfiles similares impide detectar problemas relevantes en otros tipos de usuarios.
- Introducir demasiados usuarios piloto demasiado pronto puede generar malas experiencias iniciales y provocar abandono prematuro.
- No definir una estrategia clara para aumentar el número de usuarios piloto limita la capacidad de validación del producto.
- Recoger feedback de usuarios piloto pero no mostrar cómo ha influido en el desarrollo del producto reduce su valor dentro del proyecto.
- No observar el comportamiento real de los usuarios mientras utilizan la aplicación impide detectar problemas de uso que los usuarios no siempre expresan explícitamente.
- No tener en cuenta el tiempo necesario para procesar el feedback de usuarios piloto puede afectar a la planificación de los sprints.
